\# Security Policy



\## Supported Versions



At the current stage of development, only the latest version of Aksara Nusantara is actively supported.



| Version | Supported |

|----------|-----------|

| Latest   | ✅ Yes |

| Older releases | ❌ No |



\---



\## Reporting a Vulnerability



If you discover a security vulnerability, please \*\*do not\*\* report it through a public GitHub Issue.



Instead, contact the project maintainers privately through GitHub or another official communication channel.



Please include:



\- A clear description of the vulnerability.

\- Steps to reproduce the issue.

\- Potential impact.

\- Suggested mitigation (if known).



\---



\## Response Process



The project maintainers will:



1\. Acknowledge the report.

2\. Investigate the issue.

3\. Fix the vulnerability if confirmed.

4\. Publish the fix in a future release.

5\. Credit the reporter when appropriate (unless anonymity is requested).



\---



\## Responsible Disclosure



We kindly ask security researchers to practice responsible disclosure.



Please allow sufficient time for the vulnerability to be investigated and fixed before publicly disclosing the issue.



\---



\## Scope



This policy currently applies to:



\- Desktop application

\- Source code

\- Project documentation

\- Future official releases



\---



\## Thank You



We sincerely appreciate everyone who helps improve the security and reliability of Aksara Nusantara.



Your contributions help make this project safer for everyone.
