# Development Setup

Clone the repository.

```bash
git clone https://github.com/AksaraNusantara/aksara-desktop.git
```

Enter the project directory.

```bash
cd aksara-desktop
```

Create a virtual environment.

```bash
python -m venv .venv
```

Activate the environment.

Windows

```bash
.venv\Scripts\activate
```

Linux / macOS

```bash
source .venv/bin/activate
```

Install dependencies.

```bash
pip install -r requirements.txt
```

Run the application.

```bash
python main.py
```

---

# Branch Naming

Please create a dedicated branch for your work.

Examples

```
feature/add-balinese-script

feature/improve-quiz

fix/transliteration-bug

docs/update-readme

refactor/alphabet-module
```

Avoid committing directly to the main branch.

---

# Commit Message Convention

We follow a simple commit convention.

Examples

```
feat: add alphabet learning module

fix: correct transliteration bug

docs: update README

refactor: simplify quiz logic

style: improve code formatting

test: add quiz unit tests

chore: update dependencies
```

# Pull Requests

Before submitting a Pull Request, please ensure that:

- The application runs correctly.
- New code follows the existing coding style.
- Documentation is updated when necessary.
- Commit messages follow the project convention.

Please provide a clear description of what your Pull Request changes.

---

# Coding Style

General guidelines:

- Follow PEP 8.
- Write readable code.
- Prefer descriptive variable names.
- Keep functions focused on a single responsibility.
- Add comments only when they improve understanding.

---

# Reporting Bugs

Please include:

- Operating System
- Python version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if available)

---

# Suggesting Features

Feature requests are always welcome.

Please explain:

- What problem does it solve?
- Why is it useful?
- Possible implementation ideas (optional)
